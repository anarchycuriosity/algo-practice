# 填涂颜色 — Segmentation Fault 调试教学

## 问题回顾

**题目来源：** 洛谷 P1162 填涂颜色

**题意：** 给定一个 N×N 的方阵，由 0 和 1 组成。1 围成一个闭合圈，圈内的 0 需要改成 2，圈外的 0 保持为 0。

**原代码思路（正确）：** 逆向思维——从四个角出发做 BFS，把能到达的 0 标记为 -1（圈外），最后剩下没被标记的 0 就是圈内，改成 2。

---

## Bug 1：越界导致 Segfault（致命）

### 现象

程序运行时直接崩溃，没有任何输出，典型段错误。

### 定位方法

**方法一：打印法（最快）**

在关键位置插入 `std::cerr` 输出坐标值：

```cpp
while (!q.empty())
{
    int curx = q.front().first;
    int cury = q.front().second;
    q.pop();
    for (int i = 0; i < 4; i++)
    {
        int next_x = curx + move1[i];
        int next_y = cury + move2[i];
        std::cerr << "checking: (" << next_x << ", " << next_y << ")\n";  // 调试输出
        if (next_x >= n || next_y >= n || area[next_x][next_y] == 1)
        {
            continue;
        }
        area[next_x][next_y] = -1;
        q.push({next_x, next_y});
    }
}
```

你会发现输出中有负坐标，比如 `(-1, -1)`，接着程序崩溃——说明问题出在数组访问 `area[-1][-1]`。

**方法二：GDB 调试（最精准）**

```bash
g++ -g -o debug 01_填涂颜色.cpp
gdb ./debug
(gdb) run < input.txt
# 崩溃后：
(gdb) bt          # 查看调用栈，找到崩溃位置
(gdb) frame 0     # 切换到崩溃帧
(gdb) p next_x    # 打印变量值
(gdb) p next_y
```

你会发现 `next_x` 或 `next_y` 是负数。

### 根因

```cpp
int move1[] = {1, 1, -1, -1};
int move2[] = {-1, 1, -1, 1};

// ...

if (next_x >= n || next_y >= n || area[next_x][next_y] == 1)
//  ↑ 只检查了上界，没有检查下界 (< 0)
{
    continue;
}
area[next_x][next_y] = -1;  // ← 当 next_x = -1 或 next_y = -1 时，这里直接炸
```

从 (0, 0) 出发，move1[2]=-1, move2[2]=-1，得到 (-1, -1)。`-1 >= n` 为 false，跳过 continue，直接访问 `area[-1][-1]` → **Segmentation Fault**。

### 修复

```cpp
if (next_x < 0 || next_x >= n || next_y < 0 || next_y >= n || area[next_x][next_y] == 1)
//  ↑ 加上下界检查
```

### 知识点

> **Segfault 最常见原因：数组越界访问。** 在 C++ 中，访问 `arr[-1]` 或 `arr[超出范围]` 不会报编译错误，而是运行时直接崩溃。**写 BFS/DFS 时，边界检查必须同时检查上下界四个方向。**

## Bug 2：方向数组错误（逻辑 Bug）

### 现象

修好 segfault 后，输出结果不对——圈内圈外标记混乱。

### 根因

原代码用的是一个奇怪的**对角移动**组合：

```cpp
int move1[] = {1, 1, -1, -1};   // x 方向
int move2[] = {-1, 1, -1, 1};   // y 方向
// 组合：(1,-1) (1,1) (-1,-1) (-1,1)  ← 四个对角线
```

这四个方向分别是：右下、右上、左上、左下——全是斜的。对于网格 Flood Fill，相邻格子应该是**上下左右**四个方向，斜着走会让 BFS "穿过" 只有对角接触的墙。

```
正确的 4 邻域：          错误的对角移动：
    ↑                       ↖   ↗
  ← · →                       ·
    ↓                       ↙   ↘
```

### 修复

```cpp
int dx[] = {0, 0, 1, -1};   // 右、左、下、上
int dy[] = {1, -1, 0, 0};
```

### 知识点

> **方向数组必须和算法语义匹配。** Flood Fill / BFS 走迷宫使用 4 邻域（上下左右）或 8 邻域（加上四个角）。选错方向数组会导致"穿墙"，结果完全错误。

## Bug 3：起点未标记（潜在无限循环）

### 根因

```cpp
q.push({0, 0});
q.push({0, n - 1});
q.push({n - 1, n - 1});
q.push({n - 1, 0});
// ← 四个角直接入队，但没有标记 area[角] = -1
```

虽然这个 bug 在当前数据下不一定暴露（因为方向数组的 Bug 2 掩盖了它），但如果起点是 `0`，后续 BFS 可能从其他路径再次访问到起点，造成重复入队。轻则效率低，重则队列爆掉。

### 修复

```cpp
// 入队前先标记
if (area[0][0] == 0) { area[0][0] = -1; q.push({0, 0}); }
if (area[0][n-1] == 0) { area[0][n-1] = -1; q.push({0, n-1}); }
if (area[n-1][0] == 0) { area[n-1][0] = -1; q.push({n-1, 0}); }
if (area[n-1][n-1] == 0) { area[n-1][n-1] = -1; q.push({n-1, n-1}); }
```

### 知识点

> **BFS 入队和标记必须是原子操作。** 标记 visited 的时机是「放入队列时」，不是「从队列取出时」。如果取出时才标记，同一个节点可能被多条路径同时入队，造成大量重复。

## Bug 4：只从四个角出发不够（覆盖不全）

### 根因

题目保证 1 围成一个圈，但不保证圈一定不碰边界。如果圈紧贴边界，四个角可能全是 1（墙壁），BFS 根本启动不了。

更稳健的做法是**从整个边界的所有 0 出发**。

### 修复

```cpp
for (int i = 0; i < n; i++)
{
    if (area[0][i] == 0)  { area[0][i] = -1; q.push({0, i}); }
    if (area[n-1][i] == 0) { area[n-1][i] = -1; q.push({n-1, i}); }
    if (area[i][0] == 0)  { area[i][0] = -1; q.push({i, 0}); }
    if (area[i][n-1] == 0) { area[i][n-1] = -1; q.push({i, n-1}); }
}
```

### 知识点

> **边界情况要考虑周全。** 不能假设圈一定在内部。ACM 题目中往往有边界数据来卡这种漏洞。

## Debug 心得总结

| 步骤 | 操作 |
|---|---|
| 1 | **复现** — 用题目样例确认确实崩溃 |
| 2 | **缩小范围** — 二分注释法缩小崩溃代码段 |
| 3 | **打印变量** — `cerr` 输出可疑变量值，看有没有异常值（负坐标、超大值） |
| 4 | **静态分析边界** — 检查所有数组访问，确认每个索引的范围 |
| 5 | **检查语义** — 确认每个数组/变量的含义和预期值是否一致 |

## 修正后的完整代码

```cpp
// 洛谷 P1162 填涂颜色
// 思路：从边界出发 BFS，能到达的 0 标记为 -1（圈外）
//       最后剩下的 0 就是圈内，改成 2
#include <iomanip>
#include <iostream>
#include <queue>

int area[30][30];
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

int main()
{
    int n;
    std::cin >> n;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            std::cin >> area[i][j];

    std::queue<std::pair<int, int>> q;

    // 整个边界的 0 入队（不仅是四个角）
    for (int i = 0; i < n; i++)
    {
        if (area[0][i] == 0)      { area[0][i] = -1;      q.push({0, i}); }
        if (area[n - 1][i] == 0)  { area[n - 1][i] = -1;  q.push({n - 1, i}); }
        if (area[i][0] == 0)      { area[i][0] = -1;       q.push({i, 0}); }
        if (area[i][n - 1] == 0)  { area[i][n - 1] = -1;   q.push({i, n - 1}); }
    }

    while (!q.empty())
    {
        int curx = q.front().first;
        int cury = q.front().second;
        q.pop();
        for (int i = 0; i < 4; i++)
        {
            int nx = curx + dx[i];
            int ny = cury + dy[i];
            // 四个方向都要检查
            if (nx < 0 || nx >= n || ny < 0 || ny >= n || area[nx][ny] != 0)
                continue;
            area[nx][ny] = -1;
            q.push({nx, ny});
        }
    }

    // 圈内的 0 → 2
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            if (area[i][j] == 0)
                area[i][j] = 2;

    // 输出
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            if (area[i][j] == -1)
                std::cout << std::setw(3) << 0;
            else
                std::cout << std::setw(3) << area[i][j];
        }
        std::cout << "\n";
    }
}
```
